# AI家庭教师 - 合规上架检查清单

状态: 上架前执行清单。此清单不能替代法律意见, 每次上架、重大版本更新、接入新 SDK 或改变数据处理目的前都应重新检查。

## 1. 主体与基础资质

- [ ] 明确运营主体名称、统一社会信用代码、注册地址、联系方式。
- [ ] 确认 App 名称、图标、宣传语、截图和功能描述不夸大学习效果。
- [ ] 评估是否需要 ICP 备案、App 备案、算法相关备案或其他行业资质。
- [ ] 明确目标用户年龄段和是否进入 Apple Kids Category / Google Families 相关范围。
- [ ] 准备审核账号、测试账号、演示路径和后台说明。

## 2. 协议与政策

- [ ] 用户首次使用前可查看用户协议和隐私政策。
- [ ] 隐私政策在收集个人信息前展示, 且不默认勾选同意。
- [ ] 登录/注册前记录用户协议、隐私政策、未成年人使用说明的同意版本和时间。
- [ ] 用户协议说明 AI 仅作学习辅助, 不替代教师、教材或官方答案。
- [ ] 隐私政策列明收集信息类型、处理目的、保存期限、共享对象和用户权利。
- [ ] 准备未成年人使用说明和不满十四周岁用户的专门个人信息处理规则。
- [ ] 产品内提供撤回同意、导出数据、删除数据、注销账号和投诉反馈入口。

## 3. 未成年人保护

- [ ] 一个登录用户只对应一个学生, 学习数据按账号隔离。
- [ ] 不满十四周岁用户处理个人信息前取得监护人同意。
- [ ] 不因拒绝非必要个人信息处理而拒绝基础学习辅导功能。
- [ ] 对不适合未成年人的内容进行输入和输出拦截。
- [ ] 默认不支持陌生人社交、公开评论、私信或可被其他用户发现的个人主页。
- [x] 提供复习提醒关闭入口、夜间推送限制偏好、服务端设备 token/推送投递记录和原型使用时长/休息提示；App 端需接系统级时长能力。
- [ ] 真实推送接入后验证系统通知关闭、退订、夜间免打扰和未成年人不过度打扰。
- [ ] 建立未成年人付费异常申诉和退款处理流程。

## 4. 权限与 SDK

- [ ] 相机、相册、麦克风、通知等权限均在具体功能触发时申请。
- [ ] 权限弹窗说明具体用途, 拒绝非必要权限后仍可使用文本提问等基础功能。
- [ ] 建立 SDK 清单, 包含名称、公司、版本、用途、收集信息、隐私政策链接。
- [ ] 移除与核心学习功能无关的广告追踪、设备指纹或过度统计 SDK。
- [ ] 第三方服务合同或数据处理协议覆盖保密、安全、删除、审计和分包限制。
- [ ] App Store 隐私标签和 Google Play Data safety 信息与实际数据流一致。

## 5. 数据安全

- [ ] 全站 HTTPS, 禁止明文传输账号、题目图片、AI 对话和学习记录。
- [ ] 题目图片上传生产环境已设置 `UPLOAD_STORAGE_PROVIDER=http`, 并通过 `UPLOAD_STORAGE_ENDPOINT` 接入对象存储或预签名上传网关; 当前内测本地存储已通过 `UPLOAD_RETENTION_DAYS` 和 `npm run uploads:cleanup` 明确保存期限和清理策略。
- [ ] AI 日志和过期 session 已配置 `AI_EVENT_RETENTION_DAYS`、`EXPIRED_SESSION_RETENTION_DAYS`, 并通过 `npm run retention:cleanup` 或平台定时任务定期清理。
- [ ] 验证码记录已配置 `AUTH_OTP_RETENTION_DAYS`, 生产环境关闭 `AUTH_OTP_DEV_MODE`, 并通过 `AUTH_OTP_DELIVERY_PROVIDER=http` 接入真实发送网关。
- [ ] 数据库生产账号最小权限, 管理后台强认证。
- [ ] 敏感字段加密或脱敏, 日志避免保存无必要的完整个人信息。
- [ ] 用户数据导出、删除、账号注销、备份清理流程可执行。
- [ ] AI 请求日志设置保存期限, 支持按账号定位和删除。
- [ ] 建立数据泄露、账号异常和模型安全事件应急预案。

## 6. AI 安全与教育质量

- [ ] Prompt 明确禁止直接代写、考试作弊和危险内容。
- [ ] 答疑默认采用启发式提示, 对最终答案输出设置约束。
- [ ] 变式题生成需记录原题知识点、难度和质量校验结果。
- [ ] 建立 AI 评测集, 覆盖泄题率、错因具体率、变式一致率和内容安全。
- [ ] 管理后台可查看 AI 失败、用户反馈和高风险内容。
- [ ] 重大 AI 错误可追踪到模型、Prompt 版本、输入和输出摘要。

## 7. 支付与商业化

- [ ] 免费次数、订阅权益、价格、续费规则和取消方式清晰展示。
- [ ] iOS 数字内容遵守 IAP 规则, Android 遵守对应渠道支付规则。
- [ ] 支付前有监护人确认和退款须知确认, 服务端订单状态与订阅权益一致。
- [ ] 支付回调有签名校验, 退款/取消订阅后权益同步降级。
- [ ] 取消自动续订后, 已付费周期内权益和到期时间展示一致。
- [ ] 运营后台可执行订单/订阅对账, 异常有人工处理流程。
- [ ] 未成年人付费前设置监护人确认或合理校验。
- [ ] 提供退款、发票、客服和争议处理流程。
- [ ] 不通过夸大成绩提升、制造焦虑等方式诱导付费。

## 8. 应用市场材料

- [ ] iOS: 隐私标签、年龄分级、审核账号、测试说明、付费说明、权限说明。
- [ ] Android: Data safety、目标用户年龄、Families 声明、权限说明、隐私政策链接。
- [ ] 国内安卓市场: 隐私合规检测、SDK 清单、权限说明、备案材料。
- [ ] 截图和宣传文案真实反映功能, 不承诺提分、包会、替代教师。
- [ ] 上架包与隐私政策、权限使用、SDK 清单保持一致。

## 9. 内测放量门槛

- [ ] `npm run smoke:api` 在真实数据库环境通过。
- [ ] `npm run eval:ai -- --output evals/reports/latest.json` 在目标模型上通过。
- [ ] 用户协议、隐私政策、未成年人使用说明已完成内测版审核。
- [ ] 反馈入口、删除请求入口、管理员查看反馈入口可用。
- [ ] 内测用户范围、邀请方式、风险提示和问题响应人明确。

## 10. 正式上线前门槛

- [ ] 法务审核完成并留档。
- [ ] 第三方 SDK 清单和数据流图完成。
- [ ] 监护人同意和不满十四周岁专门规则完成。
- [ ] 数据删除、注销、撤回同意链路通过测试。
- [ ] 内容安全、AI 评测、崩溃监控、成本监控和应急预案均已演练。

## 11. 参考依据

上线前需按最新官方规则复核, 当前草案主要对照:

- 《中华人民共和国个人信息保护法》: 不满十四周岁未成年人个人信息、监护人同意、专门处理规则、个人信息权利。
- 《未成年人网络保护条例》: 未成年人网络内容安全、非必要个人信息、监护人参与、访问权限控制。
- 《网络数据安全管理条例》: 网络数据处理、个人同意、敏感个人信息和未成年人个人信息处理要求。
- 《App违法违规收集使用个人信息行为认定方法》及网信办通报案例: 首次运行隐私提示、SDK 清单、必要原则、注销入口。
- Apple App Store Review Guidelines 和 Google Play Families Policies: 应用市场隐私、安全、儿童/家庭类应用和付费材料要求。
