# AI家庭教师 - 部署前检查清单

目标: 在进入小规模内测或正式 App 上线前, 用可重复的命令确认服务、数据、AI、隐私和运营链路处于可放量状态。

## 1. 代码质量闸口

- 本地执行 `cd server && npm run verify:static` 通过。
- 本地执行 `cd server && npm run schema:contract` 通过, 确认核心业务表、字段、枚举、索引和用户数据级联关系完整。
- 本地执行 `cd server && npm run api:contract` 通过, 确认前后端共享的接口合同与后端真实路由一致。
- 本地执行 `cd server && npm run frontend:contract` 通过, 确认 mobile、H5 原型和 admin 控制台的关键 API 调用、页面入口和鉴权策略没有偏离后端合同。
- 本地执行 `cd server && npm run core:contract` 通过, 确认登录、提问、AI 引导、错题、复习、报告主链路在数据库表、API、移动端页面、H5 原型和 smoke 中都有覆盖。
- 本地执行 `cd server && npm run smoke:contract` 通过, 确认 `verify:db` 会覆盖账号、学习主链路、安全拦截、Plus/支付、推送提醒、运营后台和隐私/账号删除。
- 本地执行 `cd server && npm run ci:contract` 通过, 确认 GitHub Actions 主干质量门禁完整。
- 本地执行 `cd server && npm run env:contract` 通过, 确认环境变量模板、安全默认值和部署配置检查完整。
- 本地执行 `cd server && npm run deploy:contract` 通过, 确认 Dockerfile、内测/生产环境模板和部署说明完整。
- 本地执行 `cd server && npm run stack:contract` 通过, 确认 `compose.yaml` 可同时描述 PostgreSQL 和后端 API 本地栈。
- 本地执行 `cd server && npm run stage:status` 查看阶段门禁汇总, 确认没有 `missing` 项。
- 阶段汇报前执行 `cd server && npm run stage:report`, 输出产品/项目管理视角的 ready 项、阻塞项、配置缺口和下一步命令。
- 本地执行 `cd server && npm run release:contract` 通过, 确认移动端内测包配置、权限、图标和合规基础材料完整。
- 本地执行 `cd server && npm run provider:contract` 通过, 确认 OCR、验证码、对象存储、支付和推送的真实服务商替换点完整。
- 本地执行 `cd server && npm run runtime:check` 确认 Docker/PostgreSQL 和 `DATABASE_URL` 已就绪。
- 本地执行 `cd server && npm run db:start:local`, 若 Docker 可用则启动 compose PostgreSQL 并等待端口可达; 若 Docker 不可用则按输出选择 Docker Desktop、Homebrew PostgreSQL 或远程 PostgreSQL。
- 数据库可用环境先执行 `cd server && npm run db:doctor` 通过, 确认 `DATABASE_URL`、迁移文件和 PostgreSQL TCP 端口可用。
- 数据库可用环境执行 `cd server && npm run verify:db` 通过。
- `cd server && npm run prototype:check` 通过, 确认 H5/PWA 内测壳可安装配置完整。
- `cd server && npm run mobile:check` 通过, 确认 Expo App 骨架、移动端 API 契约、拍照识题、多轮引导、错题入库、知识图谱、推送 token 注册、错题本、报告和订阅入口完整。
- CI 中 `Server Static Verification` 通过。
- CI 中 `Server API Smoke` 通过, 确认 PostgreSQL、迁移、种子数据、清理任务、提醒 dry run 和核心 API 链路可运行。
- `server/evals/reports/latest.json` 是最新一次评测输出, 且 `summary.pass` 为 `true`。
- 本地执行 `cd server && npm run ai:contract` 通过, 确认 AI 上线证据链完整。
- 真实模型环境执行 `cd server && npm run ai:check` 通过, 且输出 provider 不是 `mock`。
- Prisma migration 已提交, 不依赖手工改表。

## 2. 环境变量

进入内测前执行:

```bash
cd server
npm run deploy:contract
npm run deploy:check -- --profile internal
```

正式上线前执行:

```bash
cd server
npm run deploy:contract
npm run deploy:check -- --profile production
```

内测和生产环境变量模板位于 `deploy/internal.env.example` 和 `deploy/production.env.example`; 复制到部署平台密钥管理后必须替换所有占位值, 不要提交真实 `.env` 文件。

内测环境必须设置:

- `DATABASE_URL`: 指向内测 PostgreSQL。
- `BODY_LIMIT_BYTES`: 至少 7340032, 建议 8388608, 用于容纳 5MB 图片转 base64 后的 JSON 请求。
- `SECURITY_HEADERS_ENABLED=true`: 开启基础安全响应头。
- `RATE_LIMIT_ENABLED=true`, `RATE_LIMIT_WINDOW_MS`, `RATE_LIMIT_MAX`: 开启服务内基础限流; 生产仍需配置网关/WAF 限流。
- `CORS_ALLOWED_ORIGINS`: 目标 App/API 网关和管理后台来源列表, 多个来源用逗号分隔。
- `ADMIN_TOKEN`: 非默认值, 只给内部运营人员使用。
- `INTERNAL_TEST_INVITE_CODE`: 内测批次邀请码。
- `AUTH_OTP_SECRET`: 验证码哈希密钥, 生产必须独立设置。
- `AUTH_OTP_DEV_MODE=false`: 生产环境不得返回开发验证码。
- `AUTH_OTP_DELIVERY_PROVIDER=http`: 生产环境必须通过 HTTP 发送网关投递验证码, 不得为 `dev`。
- `AUTH_OTP_DELIVERY_ENDPOINT`, `AUTH_OTP_DELIVERY_TOKEN`: 真实短信、微信或其他验证码发送网关地址和鉴权 token。
- `AUTH_OTP_TTL_MINUTES`, `AUTH_OTP_MAX_ATTEMPTS`, `AUTH_OTP_MIN_INTERVAL_SECONDS`, `AUTH_OTP_RETENTION_DAYS`: 验证码有效期、最大尝试次数、重复发送间隔和记录保留周期。
- `ALLOW_LEGACY_USER_ID_AUTH=false`: 禁止通过 `x-user-id` 绕过 session。
- `DAILY_QUESTION_LIMIT` 和 `DAILY_AI_STEP_LIMIT`: 按成本预算设置。
- `PLUS_DAILY_QUESTION_LIMIT`, `PLUS_DAILY_AI_STEP_LIMIT`, `PLUS_PRICE_CENTS_MONTHLY`: Plus 订阅权益和价格。
- `PAYMENT_PROVIDER=http`: 生产环境必须通过真实支付网关创建 checkout, 不得使用 `dev`。
- `PAYMENT_PROVIDER_ENDPOINT`, `PAYMENT_PROVIDER_TOKEN`: 支付网关地址和鉴权 token。
- `PAYMENT_WEBHOOK_SECRET`: 支付回调 HMAC 签名密钥。
- `PUSH_PROVIDER=http`: 生产环境必须通过真实推送网关发送复习提醒, 不得使用 `dev`。
- `PUSH_ENDPOINT`, `PUSH_TOKEN`: Expo/APNS/FCM 或厂商推送网关地址和鉴权 token。
- `OPS_MAX_AI_FAILURE_RATE`, `OPS_MAX_DAILY_AI_COST`, `OPS_MIN_REVIEW_COMPLETION_RATE`, `OPS_MIN_AVERAGE_FEEDBACK_RATING`: 内测扩量阈值。
- `UPLOAD_RETENTION_DAYS`: 本地图片保留天数; 当前默认 30。
- `UPLOAD_STORAGE_PROVIDER=http`: 生产环境必须通过对象存储或预签名上传网关保存题目图片, 不得使用本地磁盘。
- `UPLOAD_STORAGE_ENDPOINT`, `UPLOAD_STORAGE_TOKEN`: 图片上传网关地址和鉴权 token, 网关需返回 `imageUrl`。
- `AI_EVENT_RETENTION_DAYS`: AI 调用日志保留天数; 当前默认 180。
- `EXPIRED_SESSION_RETENTION_DAYS`: 过期和已登出 session 保留天数; 当前默认 30。
- `NOTIFICATION_RETENTION_DAYS`: 通知投递记录保留天数; 当前默认 180。
- `DISABLED_DEVICE_TOKEN_RETENTION_DAYS`: 停用设备 token 保留天数; 当前默认 180。
- `ALLOW_PUBLIC_UPLOAD_ACCESS=false`: 禁止公开读取本地题目图片。
- `LLM_PROVIDER`, `LLM_MODEL`, `LLM_API_KEY`, `LLM_BASE_URL`: 真实模型配置。
- `LLM_TIMEOUT_MS`, `LLM_MAX_OUTPUT_TOKENS`: 真实模型请求超时和单次输出长度上限。
- `LLM_READY=true`: 只能在真实 `ai:check`、真实 `eval:ai` 和人工抽检通过后设置。
- `OCR_PROVIDER`, `OCR_API_KEY`, `OCR_ENDPOINT`: OCR 配置; mock 模式只允许用于联调。

正式上线前必须确认:

- 账号体系不再依赖 `POST /auth/mock-login`。
- 验证码登录走 `/auth/otp/request` 和 `/auth/otp/login`, 并已接入真实发送通道。
- 生产环境保持 `ALLOW_LEGACY_USER_ID_AUTH=false`, 关闭 `x-user-id` 兼容入口。
- 生产环境配置 `CORS_ALLOWED_ORIGINS`, 不允许使用 `*` 或留空。
- 生产环境保持 `SECURITY_HEADERS_ENABLED=true`; 仍需依赖 HTTPS、网关和平台安全策略做完整防护。
- 生产环境保持 `RATE_LIMIT_ENABLED=true`, 并根据内测流量调整阈值。
- 题目图片上传设置 `UPLOAD_STORAGE_PROVIDER=http`, 已接入对象存储或预签名上传网关。
- Plus 订阅通过 `/billing/checkout` 创建订单, `/billing/webhook` 激活订阅, `/billing/cancel` 取消续订, `/me.plan` 和配额使用真实订阅状态。
- 运营后台可通过 `/admin/billing` 查看订单/订阅, 通过 `/admin/billing/reconciliation` 检查已支付无订阅、退款后仍 active、过期未关闭等异常。
- 生产环境设置 `PAYMENT_PROVIDER=http`, `PAYMENT_READY=true`, 且支付、退款、取消订阅和对账流程已通过渠道沙盒测试。
- 生产环境设置 `PUSH_PROVIDER=http`, `PUSH_READY=true`, 且复习提醒已通过真实设备到达、系统通知关闭、退订和夜间免打扰测试。
- 全链路 HTTPS。
- `PRODUCTION_AUTH_READY=true`, `OBJECT_STORAGE_READY=true`, `HTTPS_READY=true` 只能在对应能力真实完成后设置。
- `ALLOW_MOCK_LOGIN=false`, 生产环境不暴露 `/auth/mock-login` 原型登录入口。
- `ALLOW_PUBLIC_UPLOAD_ACCESS=false`, 生产环境不公开暴露 `/uploads/images/:filename`。

## 3. 数据库和 API

在目标环境执行:

```bash
cd server
npm run ops:contract
npm run db:start:local
npm run db:doctor
npm run db:setup
npm run db:check
npm run retention:cleanup
npm run reminders:run -- --time 19:30 --dry-run
npm run ops:check -- --days 7
npm run smoke:api
npm run verify:db
```

如果使用 Docker 本地完整栈, 可先在项目根目录执行:

```bash
docker compose up -d postgres server
```

然后访问 `http://localhost:3000/health`, 再执行 `cd server && npm run verify:db`。

通过后再检查:

- `/ready` 返回 200。
- `/admin/metrics?days=7` 可访问且需要 `x-admin-token`。
- `/admin/ops-health?days=7` 返回 `recommendedAction`; 若为 `pause_expansion`, 不进入下一批内测邀请。
- `prototype/admin.html` 在内部来源可打开并加载运营数据; 未携带正确 `ADMIN_TOKEN` 时不能读取任何 `/admin/*` 数据。
- 注销账号会删除当前账号学习数据和本地题目图片。
- 移动端登录成功或恢复 session 后自动同步首页、复习、错题、周报、知识图谱、套餐和 readiness, 不需要用户逐页手动刷新。
- 网络受限环境先执行 `cd mobile && npm run start:check`, 确认 Metro 真实监听; 若失败, 保留结构化输出中的启动命令、探测端口、Expo startup log 路径、最近事件和输出尾部用于排查; 通过后再运行 `npm run start:local:offline` 进入体验演示。
- 真机或模拟器验证移动端今日复习“答对 / 仍不会”能调用 `/review-tasks/:id/answer`, 并分别推进掌握状态或重置复习周期。

## 4. AI 和 OCR

- 使用目标模型执行 `npm run ai:check` 通过, 确认真实 provider 可访问且没有回退到 mock。
- 使用目标模型执行 `npm run eval:ai -- --output evals/reports/latest.json` 通过。
- 使用 `AI评测记录模板.md` 完成人工抽检记录, 确认直接给答案、错因、变式题和内容安全均可接受。
- 抽查至少 10 道真实用户题, 确认 AI 不直接代写答案。
- OCR 供应商、费用、隐私处理说明已记录。
- 内容安全拦截覆盖作弊、危险、违法、未成年人高风险和敏感个人信息。

## 5. 隐私和未成年人保护

- 登录前必须勾选用户协议、隐私政策和未成年人使用说明。
- 当前账号只能对应一个学生档案, 不存在多学生切换。
- 支持导出当前账号数据。
- 支持注销账号并删除学习数据、会话、AI 日志、反馈和本地题目图片。
- 上传图片、AI 日志、过期 session、验证码记录都有保存期限和清理任务。
- 使用时长提醒、复习提醒和夜间免打扰设置可用。
- 推送 token 只绑定当前账号, 注销账号会删除设备 token 和通知投递记录。

## 6. 运营准备

- 内测邀请名单、邀请码和反馈入口已准备。
- 每日复盘使用 `/admin/summary`, `/admin/metrics`, `/admin/ops-health`, `/admin/content-review`, `/admin/questions`, `/admin/feedback`。
- 静态运营控制台 `prototype/admin.html` 只部署在内部可访问位置; 若随 H5 产物发布, 已用独立域名、VPN、基础认证或平台访问控制隔离, 并禁止公开索引。
- 发生 AI 错解、内容安全误判、模型异常成本时, 能定位到题目、Prompt 版本和 AI 事件。
- 达不到留存、复习完成率、AI 质量或成本门槛时暂停扩量。
