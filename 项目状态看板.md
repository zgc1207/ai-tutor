# AI 家庭教师 - 项目状态看板

更新时间: 2026-05-30

目标: 从当前原型逐步推进到可内测、可上架、可运营的成熟 App。

## 当前结论

当前项目处于 **阶段 4 App 化早期**。

已经不只是静态原型: 现在有后端 API、H5/PWA 内测壳、运营控制台和 Expo mobile 工程骨架。但还不是成熟 App, 因为真机、真实服务商、数据库 smoke、支付/IAP 和上架材料还没有完成。

## 已完成

### 后端能力

- Fastify + Prisma + PostgreSQL 架构。
- OTP 登录、session token、账号级数据隔离。
- 单账号只对应一个学生档案, 不做多学生切换。
- 提问、AI 启发式引导、错题入库、D1/D3/D7/D15 复习任务。
- 知识图谱、错题本、家长周报。
- 图片上传、OCR 抽象、内容安全拦截。
- Plus 订阅后端雏形: 套餐、订单、checkout、webhook、取消续订、订阅配额。
- 推送后端雏形: device token、复习提醒任务、通知投递记录。
- 管理后台 API: 用户、题目、AI 日志、反馈、内容审核、指标、运营健康、账单对账。
- 数据清理脚本: 上传图片、AI 日志、session、验证码、通知投递、停用设备 token。

### H5 / PWA

- 静态原型页面完整覆盖首页、提问、错题、复习、报告、我的、协议页。
- PWA manifest、service worker、离线页、移动端安装元信息。
- 静态运营控制台 `prototype/admin.html`, 可配置 API 和 `ADMIN_TOKEN` 查看核心运营数据。

### Mobile App

- `mobile/` Expo SDK 56 工程骨架。
- OTP 登录、bearer session API 客户端、SecureStore 保存 API 地址和 session token。
- 首页概览、文字/图片提问、多轮引导消息、加入错题/独立解决。

- `expo-notifications` 获取 Expo push token 并注册到 `/devices`。
- `mobile:check` 和 `verify:static` 已覆盖 mobile 静态检查。

## 未完成

### 必须完成后才能外部内测

- 启动 PostgreSQL, 执行 `cd server && npm run db:setup && npm run smoke:api`。
- 用 Expo 真机/模拟器跑起 mobile App。
- 真机验证相机、相册、通知权限、Expo push token 获取。
- 配置真实 LLM 并重新运行 `npm run eval:ai`。
- 配置真实 OCR 或明确 mock OCR 只用于内部演示。
- 设置内测环境变量: `ADMIN_TOKEN`, `INTERNAL_TEST_INVITE_CODE`, `AUTH_OTP_SECRET`, `CORS_ALLOWED_ORIGINS` 等。
- 用 `/admin/ops-health` 或 `npm run ops:check -- --days 7` 做扩量闸口。

### 必须完成后才能正式上架

- 真实短信/微信验证码发送通道。
- 真实对象存储或预签名上传网关。
- 真实推送网关: Expo/APNS/FCM 或厂商推送。
- iOS IAP / Android 渠道支付, 替换当前 dev/http checkout。
- App 包名、图标、启动屏、权限说明、商店截图和审核账号。
- TestFlight / Android 内测包。
- 真机端到端验收: 登录、拍照、OCR、答疑、错题、复习、推送、订阅、注销。
- 法务/合规审核用户协议、隐私政策、未成年人说明。

## 下一步建议

当前最应该做的不是继续堆页面, 而是把 mobile 从“源码骨架”推进到“真机可运行”。

推荐顺序:

1. `cd mobile && npm install`
2. `cd mobile && npm run start:local`
3. 用 Expo Go 扫码或模拟器验证 App 能启动。
4. 启动后端和 PostgreSQL, 跑通 OTP 登录、文字提问、拍照识题、错题、复习、报告。
5. 修真机发现的问题。

## 决策规则

- 若 mobile 真机无法稳定跑通登录、提问、错题、复习, 暂停新增功能。
- 若真实 AI 评测不达标, 暂停扩量, 优先修 Prompt 和评测集。
- 若 `/admin/ops-health` 返回 `pause_expansion`, 不进入下一批内测邀请。
- 若支付、推送、OCR 未接真实服务商, 只能算内部演示, 不能算可上线 App。
