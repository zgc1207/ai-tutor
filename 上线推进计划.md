# AI家庭教师 - 上线推进计划

目标: 从当前 HTML 原型推进到可上线的 iOS/Android App, 先验证学习闭环和 AI 效果, 再投入 App 化和规模化能力。

## 当前基线

- 日常进展入口见 `项目状态看板.md`; 该文件记录当前完成项、未验证项和下一步优先级。
- 当前形态: 纯 HTML/CSS/JS 移动端原型。
- 已完成: 登录、本地用户、首页、AI 答疑模拟、错题本、知识图谱、复习任务、变式题答题流程。
- 主要缺口: 真实 AI、真实存储、账号级数据隔离、OCR、后台管理、付费、推送、合规与上架材料。

## 推进原则

1. 先验证核心学习闭环, 不急着做完整 App 壳。
2. 每个阶段都有可演示产物、验收标准和是否进入下一阶段的判断。
3. 真实 AI 接入后, 优先评估答疑质量、变式题质量和错题归因质量。
4. 用户数据、未成年人保护和内容安全从阶段 2 开始纳入设计, 不能等上线前补。

## 阶段 1: 原型可验证版

周期: 3-5 天

目标: 让目标用户能完整体验一次“提问 -> 启发式答疑 -> 加入错题 -> 复习变式题 -> 知识点状态变化”的闭环。

任务:

- 修正现有假数据和交互明显错误。
- 补齐账号级数据隔离的原型表达, 一个登录用户只对应一个学生。
- 打磨 4 个核心页面: 首页、答疑、错题本、复习。
- 增加一份用户访谈记录模板, 用于收集学生/家长反馈。

验收标准:

- 手机浏览器可连续演示 5 分钟, 不出现明显流程断点。
- 至少完成 3 名学生或家长的原型试用反馈。
- 明确保留、删除和调整的功能清单。

## 阶段 2: H5 + 真实 AI MVP

周期: 2-3 周

目标: 验证 AI 真的能稳定完成启发式答疑、错因归纳和变式题生成。

技术选型:

- 前端: React + Vite 或继续静态页小步迁移。
- 后端: Node.js + Fastify/NestJS 或 Python FastAPI。
- 数据库: PostgreSQL, 后续可直接扩展 pgvector。
- AI: 先封装统一 LLM Provider, 支持模型替换。
- OCR: 先接第三方 OCR, 数学公式识别单独评估。

核心表:

- users: 账号与家长信息。
- student_profiles: 学生档案、年级、学段。一个用户对应一份学生档案。
- subjects: 科目。
- questions: 原题、输入类型、OCR 文本、AI 解析。
- error_records: 错题、错因、知识点、掌握状态。
- knowledge_nodes: 科目知识树节点。
- review_tasks: 复习计划、周期、结果。
- ai_events: AI 请求、模型、耗时、成本、失败原因。

关键能力:

- 启发式答疑 Prompt 与输出结构化。
- 错题自动归因和知识点挂载。
- 变式题生成与质量校验。
- 用户维度数据隔离, 所有学习数据必须挂到 user_id。
- 基础内容安全和敏感输入拦截。

验收标准:

- 真实题目答疑 50 题, 直接泄露答案率低于 10%。
- 变式题人工抽检 50 题, 知识点一致率达到 80% 以上。
- 错题能自动生成 D1/D3/D7/D15 复习任务。
- 单次答疑成本和响应时间可被记录。

## 阶段 3: 小规模内测版

周期: 3-5 周

目标: 支持 20-100 名真实用户连续使用, 找到留存和学习效果问题。

任务:

- 增加账号体系、短信/验证码或微信登录。`已补 /auth/otp/request 和 /auth/otp/login, 支持 dev 验证码和 HTTP 发送网关; 保留 INTERNAL_TEST_INVITE_CODE 控制 20-100 人测试准入`
- 增加后台管理: 用户、题目、AI 日志、反馈、内容审核。`已新增题目检索 /admin/questions、内容审核队列 /admin/content-review、单题回放 /admin/questions/:id 和静态运营控制台 prototype/admin.html`
- 增加学习报告: 每周错题数、薄弱知识点、复习完成率。`已新增 /reports/parent-weekly, 输出家长摘要、风险判断和行动建议`
- 增加监控: API 错误率、AI 成本、响应时间、留存。`已新增 /admin/metrics 和 /admin/ops-health, 可统计活跃、留存、复习完成率、反馈评分、AI 失败率、成本趋势, 并给出是否暂停扩量的 recommendedAction`
- 明确付费雏形: 免费次数、订阅权益、家长报告。`已新增 /plans、/billing/checkout、/billing/webhook、/billing/cancel、/billing/status 和 /me.plan; 订单/订阅会真实影响 Plus 配额, 生产仍需接入真实支付渠道`

验收标准:

- 7 日留存、复习完成率、答疑满意度可统计。`/admin/metrics?days=7 已覆盖`
- AI 失败可追踪, 关键错误能回放。`/admin/content-review 可看 AI 失败, /admin/questions/:id 可回放单题答疑链路`
- 家长能看懂学习报告, 学生愿意继续提问。`家长摘要已接入报告页`

## 阶段 4: App 化与上架准备

周期: 4-8 周

目标: 打包 iOS/Android, 完成内测分发和应用商店上架准备。

技术选型:

- App: React Native + Expo。
- 推送: Expo Notifications 或厂商推送。
- 本地缓存: SQLite/AsyncStorage。
- 支付: iOS IAP、Android 对应渠道支付。

任务:

- 迁移核心页面到 React Native。
- 建立 Expo 移动端工程骨架。`已新增 mobile/ Expo SDK 56 骨架, 包含 OTP 登录、bearer session API 客户端、安全存储、拍照/相册识题、多轮引导、错题入库、知识图谱、复习推送 token 注册、首页/提问/复习/错题/报告/Plus/我的入口和 mobile:check 静态验证`
- 在 RN 前先补可安装 H5 内测壳。`已新增 PWA manifest、service worker、离线页、移动端安装元信息和 prototype:check, 可用 HTTPS/localhost 作为阶段 3 前的轻量分发形态`
- 接入相机、相册、推送、离线缓存。`已先补 /me/reminder、DeviceToken、Push Provider、NotificationDelivery 和 /notifications/review-reminders/run, 支持提醒开关、提醒时间、夜间免打扰、设备 token、模拟/HTTP 推送和投递记录; mobile 已接入 expo-image-picker、expo-notifications 和 expo-secure-store; 原型已加入使用时长和休息提示`
- 完成隐私政策、用户协议、儿童/未成年人保护说明。
- 完成应用商店素材、审核账号、备案/合规检查。
- 按合规上架检查清单完成权限、SDK、数据安全、未成年人保护和应用市场材料核对。

验收标准:

- TestFlight/Android 内测可安装。
- 拍照答疑和复习推送可用。
- 崩溃率、接口错误率和 AI 成本在可接受范围。
- 上架材料齐备。

## 阶段 5: 正式上线与增长

周期: 持续迭代

目标: 上线后围绕真实使用数据优化学习效果、AI 成本和转化。

任务:

- A/B 测试复习周期和变式题难度。
- 引入 RAG/向量检索降低长记忆 Token 成本。
- 优化学段差异化话术。
- 增加家长端订阅、学习周报、薄弱点专题训练。
