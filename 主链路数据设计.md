# AI 家庭教师 - 主链路数据设计

更新时间: 2026-06-06

目标: 用产品经理能验收的方式说明“前端页面 -> 后端接口 -> 数据库表 -> 验收命令”的对应关系。当前版本先聚焦内测必须跑通的主学习链路, 不把支付、推送、运营后台当成第一优先级。

## 当前结论

数据库不是空白状态。当前 Prisma schema 已经覆盖第一版核心业务表: `User`, `StudentProfile`, `Session`, `AuthOtp`, `Subject`, `KnowledgeNode`, `Question`, `AnswerSession`, `AnswerMessage`, `ErrorRecord`, `ReviewTask`, `AiEvent`。

现在最大的风险不是“没有表”, 而是还没有在真实 PostgreSQL 上跑通 `verify:db`, 所以数据库迁移、种子数据和 API smoke 仍缺少本机或 CI 之外的实跑证据。

## 主链路 1: 登录和学生档案

产品动作:

- 用户输入手机号和验证码登录。
- 首次登录创建唯一学生档案。
- 后续所有学习数据按当前 session 绑定到同一个 `userId`。

前端入口:

- Mobile: 登录页、我的页。
- H5 原型: `prototype/login.html`, `prototype/me.html`。

后端接口:

- `POST /auth/otp/request`
- `POST /auth/otp/login`
- `POST /auth/logout`
- `GET /me`
- `PATCH /me/profile`

数据库表:

- `User`: 账号、手机号、昵称、协议同意版本和时间。
- `AuthOtp`: 验证码 hash、过期时间、尝试次数、消费时间。
- `Session`: bearer token、过期时间、注销时间。
- `StudentProfile`: 年级、学段、目标科目、复习提醒偏好。

验收证据:

- 静态: `cd server && npm run core:contract`
- 真实数据库/API: `cd server && npm run verify:db`

## 主链路 2: 提问和 AI 启发式答疑

产品动作:

- 学生输入文字题目或上传图片题目。
- 后端创建题目记录。
- AI 逐步给出分析、知识点、提示和最终解法。
- 每一次 AI 调用需要记录 provider、model、token、耗时、成本和失败原因。

前端入口:

- Mobile: 提问页、图片识题入口。
- H5 原型: `prototype/ask.html`。

后端接口:

- `POST /uploads/images`
- `POST /ocr/extract`
- `POST /questions`
- `POST /questions/:id/answer/next`

数据库表:

- `Subject`: 科目。
- `KnowledgeNode`: 知识点树。
- `Question`: 原题、图片地址、OCR 文本、题目状态。
- `AnswerSession`: 单题答疑会话、当前步骤、是否展示最终答案。
- `AnswerMessage`: 学生/AI 消息、消息类型、结构化 payload。
- `AiEvent`: AI 调用审计、成本和质量排查证据。

验收证据:

- 静态: `cd server && npm run core:contract && npm run ai:contract`
- 真实 AI: `cd server && npm run ai:check && npm run eval:ai -- --output evals/reports/latest.json`
- 真实数据库/API: `cd server && npm run verify:db`

## 主链路 3: 加入错题和错因归纳

产品动作:

- 学生结束答疑后, 如果没有独立解决或主动加入错题, 后端生成错题。
- 错题需要记录知识点、错因、掌握状态和正确连续次数。

前端入口:

- Mobile: 提问页“加入错题”、错题本页。
- H5 原型: `prototype/mistakes.html`。

后端接口:

- `POST /questions/:id/finish`
- `GET /mistakes`

数据库表:

- `ErrorRecord`: 错题、知识点、错因、掌握状态、正确连续次数。
- `Question`: 错题来源题目。
- `KnowledgeNode`: 可选绑定的知识点节点。

验收证据:

- 静态: `cd server && npm run core:contract`
- 真实数据库/API: `cd server && npm run verify:db`

## 主链路 4: 复习任务和掌握状态

产品动作:

- 错题入库后生成 D1/D3/D7/D15 复习任务。
- 学生答对变式题后推进掌握状态。
- 学生答错后重置复习周期。

前端入口:

- Mobile: 今日复习页、错题跳转复习。
- H5 原型: `prototype/review.html`。

后端接口:

- `GET /review-tasks/today`
- `GET /review-tasks`
- `POST /review-tasks/:id/answer`

数据库表:

- `ReviewTask`: 复习周期、到期时间、变式题、答题结果。
- `ErrorRecord`: 错题状态和正确连续次数。

验收证据:

- 静态: `cd server && npm run core:contract`
- 真实数据库/API: `cd server && npm run verify:db`

## 主链路 5: 首页、知识图谱和家长周报

产品动作:

- 首页展示学习概览、今日复习和最近答疑。
- 知识图谱按错题和掌握状态聚合红/黄/绿/灰。
- 家长周报输出本周结论、薄弱点和行动建议。

前端入口:

- Mobile: 首页、知识图谱、家长周报。
- H5 原型: `prototype/index.html`, `prototype/report.html`。

后端接口:

- `GET /dashboard`
- `GET /knowledge-tree`
- `GET /reports/parent-weekly`
- `GET /reports/weekly`

数据库表:

- `Question`: 学习频次和最近答疑。
- `ErrorRecord`: 薄弱点和掌握状态。
- `ReviewTask`: 复习完成率和待复习数量。
- `KnowledgeNode`: 知识图谱节点。

验收证据:

- 静态: `cd server && npm run core:contract`
- 真实数据库/API: `cd server && npm run verify:db`

## 当前阻塞

- 本机没有可连接 PostgreSQL: `localhost:5432` 当前不可达。
- 本机没有 Docker/psql, 所以还不能本地执行数据库迁移和 smoke。
- 真实 LLM API key 未配置, 只能完成 mock 评测和 provider 合同检查。
- 移动端 Metro 真机/模拟器链路还没有跑通。

## 下一步验收顺序

1. 安装 Docker Desktop 或提供远程 PostgreSQL。
2. 执行 `cd server && npm run db:start:local && npm run verify:db`。
3. 配置 `LLM_API_KEY`, 执行 `cd server && npm run ai:check`。
4. 执行 `cd server && npm run eval:ai -- --output evals/reports/latest.json`。
5. 执行 `cd mobile && npm run runtime:check && npm run start:check`。
6. 用真机或模拟器走一遍: 登录 -> 提问 -> AI 引导 -> 加入错题 -> 复习 -> 家长周报。
