# 角色与任务
你现在是顶级的产品经理与全栈架构师，擅长AI教育类产品设计。请帮我设计并开发一款名为【AI家庭教师】的移动端App。请提供从产品逻辑、数据架构到核心功能模块的完整设计方案，并给出关键代码的实现思路。

# 核心定位
一款基于大模型能力的K12个性化AI辅导App，主打“答疑+错题本+周期性巩固”的学习闭环，模拟真实优秀家庭教师的1对1辅导体验。

# 核心目标用户与场景
- 学生：需解答难题、避免反复踩坑、考前针对性复习。
- 家长（监管端）：需查看单个学生的学习进度、切换科目、掌握错题分布。

# 核心功能需求（请围绕以下功能深度展开设计）

## 1. 单学生账号与多科目体系 (身份与边界)
- **账号级数据隔离**：一个登录用户只对应一个学生，错题、知识图谱、复习任务和学习报告都绑定到该用户。
- **多科目独立管理**：每个学生账号下细分数学、物理、英语等科目，错题和知识图谱按科目构建，互不干扰。
- **学段自适应**：根据学生的年级（如初二），自动调整AI答疑的深度和语言风格（对小学生更生动，对高中生更严谨）。

## 2. 智能答疑模块 (不只是给答案)
- **苏格拉底式启发答疑**：AI绝对不能直接给出最终答案，必须采用“启发式提问”，一步步引导学生思考，得出答案。
- **解题步骤结构化**：答疑结果需包含：【题目分析】->【涉及知识点】->【分步提示】->【最终解法】->【举一反三（相似题）】。
- **多模态输入**：支持拍照识题（OCR+图像理解）、语音提问、文本输入。

## 3. 错题与知识图谱长期记忆 (核心壁垒)
- **自动归集错题**：凡是学生未独立答对、经过AI提示才解出的题目，自动进入“错题本”。
- **错题深度解析**：AI需自动提取错题的【核心卡点知识点】（如：二次函数动点问题），并生成【错误原因分析】（如：概念混淆、计算失误）。
- **知识图谱关联**：将错题知识点挂载到该科目的知识图谱树上，标记该节点为“薄弱点（红灯）”。

## 4. 周期性针对性检验 (遗忘曲线对抗)
- **艾宾浩斯复习策略**：基于遗忘曲线，在错题发生后的第1天、第3天、第7天、第15天自动推送“复习任务”。
- **变式题生成（举一反三）**：复习时绝不重做原错题，AI需根据原错题的【核心卡点知识点】和【难度】，动态生成“同知识点但不同情境”的变式题（换数字、换背景、逆向提问）。
- **通关机制**：如果变式题连续两次做对，该知识点降级为“已掌握（绿灯）”；如果再做错，重置遗忘曲线计时，重新开始周期。
