# AI 家庭教师 - 最小真实闭环执行清单

更新时间: 2026-06-06

目标: 下一阶段只证明一件事: 用户可以从页面完整走完“登录 -> 提问 -> AI 引导 -> 加入错题 -> 生成复习 -> 查看报告”。这份清单用于产品/项目视角跟进, 不再把重点放在扩展门禁脚本。

## 当前判断

当前代码已经具备主链路的结构, 但还不能宣称“最小真实闭环已跑通”。

原因:

- 后端和数据库链路需要 PostgreSQL 真实可连接后执行 `verify:db`。
- AI 链路需要配置真实 `LLM_API_KEY`, 并通过 `ai:check` 和真实评测。
- 移动端 Metro 当前受本机环境阻塞; 在修复前, 最小闭环优先使用 H5/PWA 或 CI smoke 证明后端链路。

## 只验收这 6 步

### 1. 登录

产品动作: 用户输入手机号和验证码, 获得 session。

当前代码支撑:

- API: `POST /auth/otp/request`, `POST /auth/otp/login`, `GET /me`
- 表: `User`, `AuthOtp`, `Session`, `StudentProfile`
- 前端: H5 登录页、Mobile 登录页

验收证据:

- `verify:db` 输出 `otp_request`, `login`, `me`

当前状态: 代码已具备, 等真实 PostgreSQL 运行证明。

### 2. 提问

产品动作: 用户提交一道文字题或图片题。

当前代码支撑:

- API: `POST /questions`, `POST /uploads/images`, `POST /ocr/extract`
- 表: `Question`, `Subject`
- 前端: H5 提问页、Mobile 提问页

验收证据:

- `verify:db` 输出 `question`, `ocr`, `upload_image`

当前状态: 代码已具备, 等真实 PostgreSQL 运行证明。

### 3. AI 引导

产品动作: 后端调用 AI, 返回启发式提示, 不直接泄露答案。

当前代码支撑:

- API: `POST /questions/:id/answer/next`
- 表: `AnswerSession`, `AnswerMessage`, `AiEvent`
- Provider: `server/src/ai/llm-provider.js`

验收证据:

- mock 评测: `cd server && npm run eval:ai -- --output evals/reports/latest.json`
- 真实 AI: `cd server && npm run ai:check`
- 数据库/API: `verify:db` 输出 `answer_next`

当前状态: mock 评测通过; 真实 AI key 未配置。

### 4. 加入错题

产品动作: 用户结束答疑并加入错题本。

当前代码支撑:

- API: `POST /questions/:id/finish`, `GET /mistakes`
- 表: `ErrorRecord`
- 前端: H5 错题页、Mobile 错题页

验收证据:

- `verify:db` 输出 `finish`, `mistakes`

当前状态: 代码已具备, 等真实 PostgreSQL 运行证明。

### 5. 生成复习

产品动作: 错题生成 D1/D3/D7/D15 复习任务, 用户可以提交变式题答案。

当前代码支撑:

- API: `GET /review-tasks/today`, `POST /review-tasks/:id/answer`
- 表: `ReviewTask`, `ErrorRecord`
- 前端: H5 复习页、Mobile 复习页

验收证据:

- `verify:db` 输出 `review_tasks`, `review_answer`

当前状态: 代码已具备, 等真实 PostgreSQL 运行证明。

### 6. 查看报告

产品动作: 用户或家长查看首页概览、学习报告、知识图谱。

当前代码支撑:

- API: `GET /dashboard`, `GET /reports/parent-weekly`, `GET /knowledge-tree`
- 表: `Question`, `ErrorRecord`, `ReviewTask`, `KnowledgeNode`
- 前端: H5 首页/报告页、Mobile 首页/报告/知识图谱

验收证据:

- `verify:db` 输出 `dashboard`, `parent_weekly_report`, `knowledge_tree`

当前状态: 代码已具备, 等真实 PostgreSQL 运行证明。

## 下一步执行顺序

### 第一步: 先证明后端和数据库

需要环境:

- Docker Desktop, 或本机 PostgreSQL, 或一个可访问的远程 PostgreSQL。

执行:

```bash
cd server
npm run db:start:local
npm run verify:db
```

通过标志:

- 命令成功结束。
- 输出包含 `main_flow_acceptance`。

### 第二步: 再证明真实 AI

需要环境:

- `LLM_PROVIDER`
- `LLM_BASE_URL`
- `LLM_MODEL`
- `LLM_API_KEY`

执行:

```bash
cd server
npm run ai:check
npm run eval:ai -- --output evals/reports/latest.json
```

通过标志:

- `ai:check` 成功。
- `eval:ai` 通过。
- 人工抽检记录到 `AI评测记录模板.md`。

### 第三步: 再接页面

优先路径:

- 如果 Mobile Metro 仍阻塞, 先用 H5/PWA 配后端 API 走通。
- 如果 Mobile Metro 修复, 用 Expo Go/模拟器走通。

验收路径:

```text
登录 -> 提问 -> 获取 AI 引导 -> 加入错题 -> 进入复习 -> 查看报告
```

通过标志:

- 页面上可以连续走完, 不依赖本地假数据。
- 后端有对应数据库记录。

## 当前不做

最小真实闭环跑通前, 暂停扩展以下内容:

- 新增复杂 UI 页面。
- 新增支付/IAP 细节。
- 新增推送服务商。
- 新增 OCR 服务商。
- 新增运营后台功能。
