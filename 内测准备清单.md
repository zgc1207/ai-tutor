# AI家庭教师 - 小规模内测准备清单

目标: 在阶段 2 真实 AI 和真实数据库链路跑通后, 支持 20-100 名用户连续使用, 判断是否进入 App 化阶段。

## 内测前必须完成

- PostgreSQL 数据库可用, `npm run smoke:api` 通过。
- 数据库使用 `npm run db:setup` 初始化, `npm run db:check` 通过。
- 数据库环境执行 `npm run verify:db` 通过。
- `npm run verify:static` 通过, 确认不依赖数据库的质量门槛没有回归。
- GitHub Actions 中 `Server Static Verification` 通过。
- GitHub Actions 中 `Server API Smoke` 通过。
- `npm run readiness:static` 通过或仅剩明确可接受的 warning。
- `npm run config:check` 通过, `/ready` 在内测环境返回 200。
- `npm run deploy:check -- --profile internal` 在内测环境通过。
- `BODY_LIMIT_BYTES` 已设置为至少 7340032, 避免 5MB 图片 base64 上传被框架请求体上限提前拒绝。
- `SECURITY_HEADERS_ENABLED=true` 已设置, 基础安全响应头可在 `/health` 等接口看到。
- `RATE_LIMIT_ENABLED=true` 已设置, 内测期间根据 429 记录调整 `RATE_LIMIT_WINDOW_MS` 和 `RATE_LIMIT_MAX`。
- `CORS_ALLOWED_ORIGINS` 已按内测 H5/API 网关/管理后台来源配置。
- AI 上线证据链完整, `npm run ai:contract` 通过。
- 真实模型配置完成, `npm run ai:check` 和 `npm run eval:ai -- --output evals/reports/latest.json` 通过。
- 已按 `AI评测记录模板.md` 完成人工抽检, 并确认可以进入小规模内测。
- `DAILY_QUESTION_LIMIT` 和 `DAILY_AI_STEP_LIMIT` 已按内测成本预算设置。
- `/uploads/images` 能接收题目图片并返回可用于 OCR 的 `imageUrl`; `UPLOAD_STORAGE_PROVIDER=local` 可用于开发和早期内测, 生产前必须设置 `UPLOAD_STORAGE_PROVIDER=http`、`UPLOAD_STORAGE_ENDPOINT` 和 `UPLOAD_STORAGE_TOKEN`, 接入对象存储或预签名上传网关。`UPLOAD_RETENTION_DAYS` 已设置, `npm run uploads:cleanup` 已加入每日清理任务。`AI_EVENT_RETENTION_DAYS`、`EXPIRED_SESSION_RETENTION_DAYS`、`AUTH_OTP_RETENTION_DAYS`、`NOTIFICATION_RETENTION_DAYS` 和 `DISABLED_DEVICE_TOKEN_RETENTION_DAYS` 已设置, `npm run retention:cleanup` 已加入每日数据库清理任务。`ALLOW_PUBLIC_UPLOAD_ACCESS` 只允许作为早期 OCR URL 联调开关, 生产必须关闭。
- `OCR_PROVIDER` 已按内测环境配置；mock 模式可跑通拍照链路, 接第三方 OCR 前需补充供应商、费用和隐私说明。
- `/plans` 能展示免费次数、Plus 订阅权益、价格和家长报告权益。
- `/billing/checkout`、`/billing/webhook`、`/billing/status` 能跑通开发 provider; webhook 激活后 `/me.plan` 和每日配额切换到 Plus。
- `/billing/cancel` 能取消自动续订但保留当前周期权益; 退款 webhook 会立即取消对应订阅并降级权益。
- `/admin/billing` 和 `/admin/billing/reconciliation` 可用于内测期人工核对订单、订阅和异常状态。
- 对外商业化前必须把 `PAYMENT_PROVIDER` 从 `dev` 切到真实渠道, 并完成退款、取消订阅和对账演练。
- H5 内测分发前执行 `cd server && npm run prototype:check`, 确认 PWA manifest、service worker、离线页和所有原型页安装元信息完整。
- App 化联调前执行 `cd server && npm run mobile:check`, 确认 Expo 骨架、App 配置、JS 语法、bearer session API 契约、拍照识题、多轮引导、错题入库、知识图谱、推送 token 注册、错题本、家长周报和 Plus 订阅入口没有回归。
- 使用 HTTPS 或 localhost 访问原型, 确认移动端浏览器可以添加到主屏幕; `file://` 方式不会注册 service worker。
- `INTERNAL_TEST_INVITE_CODE` 已按内测批次设置, 未受邀用户无法同步后端账号。
- `/auth/otp/request` 和 `/auth/otp/login` 已可跑通验证码登录; `AUTH_OTP_DEV_MODE=true` 只允许用于开发和小范围内测, 生产前必须设置 `AUTH_OTP_DELIVERY_PROVIDER=http`、`AUTH_OTP_DELIVERY_ENDPOINT` 和 `AUTH_OTP_DELIVERY_TOKEN`, 接入真实发送通道并关闭 dev mode。
- `ALLOW_LEGACY_USER_ID_AUTH=false` 已设置, 业务接口不再接受 `x-user-id` 早期联调入口。
- 后端登录返回 session token, 原型业务请求优先使用 bearer token, 不依赖手工传 `x-user-id`。
- 内测阶段可保留 `ALLOW_MOCK_LOGIN=true`; 进入生产前必须接真实账号体系并设置 `ALLOW_MOCK_LOGIN=false`。
- `/me/reminder` 能保存复习提醒开关、提醒时间和夜间免打扰时间段, 原型“我的”页可关闭提醒。
- `/devices` 能注册/停用当前账号的 App 或 PWA 推送 token, 不允许跨账号查看 token。
- `/notifications/review-reminders/run` 能按复习提醒时间、待复习任务和夜间免打扰规则生成投递记录; 内测 cron 可先使用 `PUSH_PROVIDER=dev` 演练。
- `npm run reminders:run -- --time 19:30 --dry-run` 能在目标环境执行; 内测时由部署平台按提醒时间槽定时运行 `npm run reminders:run -- --time HH:mm`。
- 对外 App 内测前必须将 `PUSH_PROVIDER` 接到 Expo/APNS/FCM 或厂商推送网关, 并验证 mobile 端 Expo push token 注册、系统通知关闭、退订和夜间免打扰。
- 原型能显示今日使用时长, 连续使用 25 分钟提醒休息, 当天累计 45 分钟提醒控制节奏。
- 登录前必须显式勾选用户协议、隐私政策和未成年人使用说明, 后端记录同意版本和时间。
- 原型配置后端 API 后, 登录、年级更新、文字/拍照提问、加入错题、复习任务、复习答题、错题本、知识图谱、学习报告、反馈提交均可用。
- mobile 配置后端 API 后, OTP 登录、文字/拍照提问、多轮引导、加入错题/独立解决、今日复习、错题本、知识图谱、家长周报、Plus 订阅状态和复习推送 token 注册入口可用。
- `/reports/parent-weekly` 能生成家长可读摘要、风险判断和行动建议, 报告页能展示家长摘要。
- 复习答题能真实改变错题掌握状态: 连续 2 次答对转为已掌握, 答错重置后续复习周期。
- `/admin/metrics?days=7` 能返回 7 日留存、复习完成率、反馈评分、AI 失败率和成本趋势。
- `/admin/ops-health?days=7` 或 `npm run ops:check -- --days 7` 能给出 pass/warn/fail 和 `recommendedAction`; 出现 `pause_expansion` 时暂停扩量。
- `prototype/admin.html` 能在内部网络或受控域名打开, 填入后端 API 地址和 `ADMIN_TOKEN` 后可加载扩量健康、关键指标、商业化对账、内容审核和最近用户。
- 静态运营控制台仅供内部使用; 若与 H5 一起托管, 已通过 VPN、基础认证、平台访问控制或独立内部域名限制访问, 且 `CORS_ALLOWED_ORIGINS` 未放开到不可信来源。
- `/admin/questions` 能检索内测题目, `/admin/content-review` 能查看安全拦截、AI 失败、低评分反馈和内容质量反馈, `/admin/questions/:id` 能回放关键错题答疑链路。
- 原型“我的”页可导出当前账号数据, 并可注销账号删除后端学习数据和本地题目图片。
- 提问和反馈链路已启用基础内容安全检查, `npm run eval:ai` 的内容安全用例通过。
- 至少准备 30-50 道真实评测题, 覆盖数学、物理、英语、语文。
- 完成并审核 `用户协议草案.md`、`隐私政策草案.md`、`未成年人使用说明.md`。
- 按 `合规上架检查清单.md` 完成内测放量门槛检查。
- 按 `部署前检查清单.md` 完成目标环境部署前检查。

## 进入 App 化阶段的建议门槛

- 7 日留存达到可接受水平。
- 复习任务有稳定完成行为。
- AI 评测达标, 且真实用户反馈中严重错解可控。
- 单题成本可接受。
- 学生愿意继续用, 家长能理解学习报告价值。
